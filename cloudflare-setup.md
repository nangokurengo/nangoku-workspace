# Cloudflare Pages デプロイ連携設定ガイド

GitHubリポジトリ `nangokurengo/nangoku-workspace` から Cloudflare Pages へ自動デプロイを行うための設定手順です。

---

## 1. Cloudflare ダッシュボードでのセットアップ手順

1. [Cloudflare ダッシュボード](https://dash.cloudflare.com/) にログインします。
2. 左メニューから **「Workers & Pages」** を選択します。
3. **「作成」** ボタンをクリックし、**「Pages」** タブを選択してから **「Git に接続」** をクリックします。
4. GitHubアカウントとの連携を許可し、リポジトリ **`nangokurengo/nangoku-workspace`** を選択して **「セットアップの開始」** をクリックします。

---

## 2. ビルド設定のカスタマイズ

ビルド設定画面で、以下のパラメータを入力・確認します。

- **プロジェクト名**: `nangoku-workspace` (任意)
- **プロダクションブランチ**: `main`
- **フレームワークプリセット**: `Astro`
- **ビルドコマンド**: `npm run build`
- **出力ディレクトリ**: `dist`

---

## 3. 環境変数の設定 (重要)

本プロジェクトで使用している Astro v6 は **Node.js v22.12.0 以上** を必要とします。Cloudflare Pages のデフォルトビルド環境が古い場合があるため、以下の環境変数を必ず設定してください。

1. ビルド設定画面の **「環境変数（詳細）」** アコーディオンを展開します。
2. 以下の変数を追加します。

| 変数名 (Variable Name) | 値 (Value) |
| :--- | :--- |
| **`NODE_VERSION`** | `22` |

> [!TIP]
> 既にプロジェクトを作成済みの場合は、ダッシュボードの **[設定] -> [ビルド & デプロイ] -> [環境変数]** から設定・追加することも可能です。

---

## 4. 自動デプロイの確認

設定が完了したら **「保存してデプロイ」** をクリックします。
初回ビルドが走り、成功すると公開URL（例: `https://nangoku-workspace.pages.dev`）が発行されます。

以降は、ローカルでコードを変更して `main` ブランチに `git push` するだけで、自動的にビルドと本番デプロイが実行されるようになります。
